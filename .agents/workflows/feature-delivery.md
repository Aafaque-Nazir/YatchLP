When I invoke /feature-delivery, deliver the requested feature end-to-end.

Inspect the existing codebase, patterns, dependencies, scripts, and relevant flows before editing.

Define the user outcome, affected files, data/API changes, security impact, and acceptance criteria.

Implement the smallest complete solution that matches the current stack and design language.

Include responsive UI, loading, empty, error, success, and permission states where relevant.

Validate inputs, enforce server-side authorization, protect secrets, and avoid unrelated dependencies or refactors.

Add or update appropriate tests, including regression tests for bug fixes.

Run applicable lint, typecheck, tests, and production build. Never claim an unavailable check passed.

Finish with: Result, What changed, How it works, Verification, and any real remaining risk.
